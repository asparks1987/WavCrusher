# Support

WavCrusher is currently a documentation and build-plan package; there is no supported executable yet.

Once development begins, use repository issues for reproducible bugs and feature discussions. Before public launch, add the actual repository links and maintainer contact routes rather than placeholders.

## Include in a support request

- App version/commit.
- Windows version and architecture.
- WavPack reported version and dependency-hash status.
- Operation type and failed stage/code.
- Whether the problem reproduces with a generated/small fixture.
- Redacted report/journal excerpt.
- Available disk space and whether storage is local/network/removable.

Never upload sensitive audio merely to demonstrate a filename/path issue. Use the support-bundle redaction feature when implemented.

For possible security or data-safety vulnerabilities, follow `SECURITY.md`.
